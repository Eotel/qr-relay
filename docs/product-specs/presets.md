# Presets

Last reviewed: 2026-05-20

QR Relay は **遊び方ごとに別アプリではない**。1 つの統合エンジン `relayHandler` に
ScanRule (config) を渡したものをプリセットと呼ぶ。MVP では 5 つのプリセットを内蔵。

> プリセットの追加方法は
> [../design-docs/scan-handler-contract.md](../design-docs/scan-handler-contract.md)。

## 哲学

「相手に渡すと自分から消える」「持っていない人だけが受け取れる」のような **直交する選択**
は、ゲーム名ではなく option 軸として表現する。これにより:

- 「奪い合い」「コレクション」「あいさつ」の区別は `onScan.source` と `onScan.sink` の
  組合せの違いにすぎない、ということが明示できる
- 1 人だけスタート / 全員スタート / 持点 N でスタート、のような数の調整がプリセット
  バリエーションのバリエーションを生む
- プレイヤーが直感する「ゲーム名」と内部のコードの責務が分離される

「N 秒で終わる」「N 人達成で終わる」のような自動終了は engine から外し、ホストが
ストップウォッチを見て手動で `リセット` する運用に集約した
([ADR-0002](../adr/0002-move-end-conditions-out-of-engine.md))。
タイマーが必要な遊び方は「バトン + ストップウォッチを目視して止める」、ノルマ系は
「コレクションを 10 人達成したら司会が手動で stop」のように既存プリセット + 運用に
読み替える。

## 5 プリセット

各エントリの `config` 表現は
[`packages/handlers/src/presets.ts`](../../packages/handlers/src/presets.ts) を参照。
ここでは「遊び方」を中心に書く。

### バトン
- 1 人だけがバトンを持っている状態でスタート
- 持っている人の QR を別の人がスキャンすると、バトンがその人に **移る** (元の持ち主からは消える)
- 既にバトンを持っている人はスキャンできない (受け取り側に空きが必要)
- 計測軸: 「保持中フラグ」「総スキャン数」
- **運用例 (旧ホットポテト)**: ホスト画面のストップウォッチを 60 秒見て、止まった時点で
  バトンを持っていた人が「負け」。司会が `一時停止` か `リセット` を押す

### 感染
- 1 人だけ感染者でスタート (`token`, `holders: "one"`)
- 感染者の QR をスキャンすると、スキャンした人も感染する (感染者は **減らない**、雪だるま式)
- 終了は手動: 全員感染したらホストが `リセット`、または時間で打ち切る
- 計測軸: 保持中 (= 感染者) のプレイヤー一覧と人数

### 奪い合い
- 全員が持点 10 でスタート
- スキャンすると相手から 1 点 **奪う** (相手 -1、自分 +1)
- 相手が 0 点なら奪えない (`minValue: 0`)
- 計測軸: 各プレイヤーの score
- 派生応用: 初期点数や `amount` を調整すれば、コロシアム / リレー双方に寄せられる

### コレクション
- 全員 0 点でスタート
- 誰かをスキャンすると自分に +1 (相手は変化なし)
- 同じ相手からは 1 回しかカウントされない (`uniquePerPair: true`)
- 「何人と出会えたか」を競う収集型
- **運用例 (旧ノルマ)**: ホストが「10 人達成」を目標に設定し、達成したプレイヤーが
  出た時点で司会が `リセット`

### あいさつ
- 全員 0 点でスタート
- スキャンするとお互いに +1
- 同じ相手と何回でもカウント可
- 「総あいさつ数」を稼ぐ協力 / 雑談型

## 旧プリセットの整理 (2026-05-20)

| 旧プリセット | 後継 | 説明 |
|---|---|---|
| ホットポテト | バトン + ストップウォッチ手動停止 | 60 秒タイマーは engine から外した ([ADR-0002](../adr/0002-move-end-conditions-out-of-engine.md)) |
| ノルマ | コレクション + 手動 `リセット` | 同上 |
| 鬼ごっこ | バトン | 「鬼の QR を読んだ人が鬼になる」モデルはバトンと挙動が等価 ([ADR-0001](../adr/0001-drop-status-value-kind.md)) |
| 鬼交代 | バトン | swap も「鬼の token が移動」と等価 |
| 感染 (旧 status 版) | 感染 (token 版) | `status` 軸自体を廃止、`token` で再定義 |

## バリエーションの作り方

プリセットの細部 (初期点数、対象範囲) を変えたいだけの場合、別プリセットとして
追加するか、`/new` から作成時に手動で `ScanRule` を編集する (現在は preset 選択のみ、
config 詳細編集 UI は未実装 — tech-debt-tracker 参照)。

「全く違う構造の遊び方」が欲しい場合は新 handler を追加する。詳しくは
[../design-docs/scan-handler-contract.md](../design-docs/scan-handler-contract.md) の
「relay で表現できないとき」セクション。
